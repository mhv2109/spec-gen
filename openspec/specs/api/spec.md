### Requirement: APIAuthentication

The API SHALL require authentication via: None, API key (ANTHROPIC_API_KEY, OPENAI_API_KEY, GEMINI_API_KEY, or OPENAI_COMPAT_API_KEY), API Key (ANTHROPIC_API_KEY or OPENAI_API_KEY), Not applicable (CLI command), None (uses environment variables for LLM API keys)

#### Scenario: AuthenticatedRequest
- **GIVEN** a request with valid authentication credentials
- **WHEN** the request is processed
- **THEN** the request is authenticated successfully

#### Scenario: UnauthenticatedRequest
- **GIVEN** a request without authentication
- **WHEN** accessing a protected endpoint
- **THEN** the response status is 401 Unauthorized

### Requirement: CliCommandprojectConfiguration

The API SHALL support `CLI Command spec-gen init` to initialize spec-gen in the current project by detecting project type, checking for existing configurations, creating a .spec-gen configuration file, updating .gitignore, and preparing the project for analysis.

**Request:**

```json
{
  "options": {
    "force": "boolean",
    "openspec-path": "string"
  }
}
```

**Response:**

```json
{
  "status": "string",
  "messages": "string[]",
  "config": {
    "projectType": "string",
    "openspecPath": "string",
    "createdAt": "string"
  }
}
```

#### Scenario: InitializeWithDefaults
- **GIVEN** A valid project directory with no existing spec-gen configuration
- **WHEN** The command 'spec-gen init' is executed
- **THEN** A new .spec-gen/config.json file is created, the project type is detected, and the .gitignore file
is updated if necessary
- **AND** The user is prompted to confirm adding .spec-gen/ to .gitignore if it's not already present
- **AND** A summary of the initialization process is displayed

#### Scenario: OverwriteExistingConfiguration
- **GIVEN** An existing .spec-gen/config.json file
- **WHEN** The command 'spec-gen init --force' is executed
- **THEN** The existing configuration is overwritten with a new one
- **AND** The user is not prompted to confirm overwriting
- **AND** The project type is re-detected and the .gitignore file is updated if necessary

#### Scenario: CustomOpenspecPath
- **GIVEN** A valid project directory
- **WHEN** The command 'spec-gen init --openspec-path ./docs/specs' is executed
- **THEN** The OpenSpec output directory is set to ./docs/specs
- **AND** The .spec-gen/config.json file is created with the custom OpenSpec path
- **AND** The project type is detected and the .gitignore file is updated if necessary
